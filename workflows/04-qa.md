# Workflow 04 — Local QA

## Objective
Run a complete local quality assurance check before any code is pushed or deployed.

## Inputs
- `LOCAL_URL` — local dev server URL (default: `http://localhost:3000`)
- `TARGET_PROJECT` — project folder under `projects/`

## Tools
- None (manual + browser DevTools)

## Execution Steps

1. **Start dev server**
   ```bash
   cd projects/{{TARGET_PROJECT}}
   npm run dev
   ```

2. **Run checklist**

   ### Load
   - [ ] Site loads at `{{LOCAL_URL}}` without error page
   - [ ] No unhandled runtime errors in terminal
   - [ ] No red errors in browser console

   ### Layout
   - [ ] Hero is visible above the fold
   - [ ] CTA button is visible and clickable
   - [ ] All 10 sections render in correct order
   - [ ] Text is readable (sufficient contrast)
   - [ ] Spacing and hierarchy look premium

   ### Mobile (use DevTools device emulation)
   - [ ] 375px (iPhone SE) — layout intact, text readable
   - [ ] 768px (iPad) — layout intact, no overflow
   - [ ] CTA is visible and tappable on mobile
   - [ ] Animations simplified or disabled on mobile if they cause jank

   ### Assets
   - [ ] All images load (no broken image icons)
   - [ ] All frames load (if scroll animation present)
   - [ ] No 404s in network tab

   ### Animations
   - [ ] Scroll animation plays without dropped frames
   - [ ] Entrance animations trigger correctly
   - [ ] No motion blocks readability of copy

   ### Performance (Lighthouse or DevTools)
   - [ ] LCP < 2.5s
   - [ ] No render-blocking scripts
   - [ ] Images are optimized

3. **Document failures**
   - List every failing check
   - Do NOT push until all checks pass

4. **Fix failures**
   - Use Workflow 06 (Iterate) for each issue
   - Re-run this QA after each fix

## Outputs
- QA pass/fail report (list of checks with status)
- All issues logged before moving to deploy

## QA Gate
**Do NOT proceed to Workflow 05 (Deploy) until all checks above are marked passing.**

## Notes
- QA is not optional — it is part of every build cycle
- "Looks fine" is not a pass — every checkbox must be verified
- Mobile must work — this is non-negotiable
